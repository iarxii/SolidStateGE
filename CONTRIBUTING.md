# Contributing Guidelines

First off, thank you for considering contributing to this open-source utility! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## How Can I Contribute?

### 1. Reporting Bugs
When submitting a bug report via the issue tracker, please include:
- A quick summary of the issue
- Steps to reproduce
- What you expected to happen vs what actually happened
- Environment details (OS, Node version, Python version, etc.)

### 2. Suggesting Enhancements
Enhancement suggestions are tracked as GitHub issues. Create an issue describing the feature, why it is needed, and how it should work.

### 3. Pull Requests
We actively welcome Pull Requests!
1. **Fork the repository** and create your branch from `main`.
2. **Read the README** to understand the setup and architecture.
3. If you've added code that should be tested, add tests.
4. Update documentation if necessary.
5. Ensure your code lints and builds cleanly.
6. Open a Pull Request referencing any related issues.

## Code of Conduct
Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms. Be respectful and constructive in all reviews and discussions.
